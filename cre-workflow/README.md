# Paladin Protocol — Chainlink CRE Workflow

Real Chainlink **Compute Runtime Environment (CRE)** workflow using `@chainlink/cre-sdk@1.1.3`.
Compiled to **WASM** via Bun + Javy for deployment to a Chainlink DON.

## Architecture

```
CronTrigger (every 1 min)
    │
    ▼
HTTPClient ──► Alchemy JSON-RPC ──► latest block data
    │
    ▼
Heuristic risk scoring (tx count + gas usage)
    │
    ├─ low risk ──► MONITOR (return)
    │
    ▼ suspicious
HTTPClient ──► Ollama /api/generate ──► AI exploit classification
    │            (llama3.1:8b, local or remote)
    │
    ▼
EVMClient.callContract ──► RiskRegistry.eventCount() (read)
    │
    ├─ confidence ≥ 60% AND severity ≥ 3 ──► EVMClient.writeReport()
    │                                          (CRE report → on-chain)
    │
    └─ below threshold ──► MONITOR (return)
```

## Real CRE SDK Usage

| CRE Feature | Usage in this workflow |
|---|---|
| `CronCapability` | Trigger every 1 minute |
| `HTTPClient.sendRequest` (consensus mode) | Fetch blocks from Alchemy RPC |
| `HTTPClient.sendRequest` (consensus mode) | AI analysis via Ollama |
| `EVMClient.callContract` | Read `RiskRegistry.eventCount()` |
| `EVMClient.writeReport` | Record threats via CRE report mechanism |
| `runtime.report(prepareReportRequest(...))` | Encode on-chain write payload |
| `Runner.newRunner<Config>` | Workflow entry point with config parsing |
| `consensusIdenticalAggregation` | DON-wide consensus for HTTP responses |

## Compilation

Requires [bun](https://bun.sh) runtime:

```bash
# One-time setup (downloads Javy WASM compiler)
bun x cre-setup

# Compile TypeScript → WASM
bun x cre-compile index.ts
# Output: index.wasm (3.8 MB)
```

Or via npm:
```bash
npm run compile
```

## Simulation

Requires the [CRE CLI](https://github.com/smartcontractkit/cre-cli) and a Chainlink account:

```bash
# Login to your Chainlink CRE account
cre login

# Run local simulation (triggers your 1-min cron once)
cre workflow simulate . --trigger-index 0

# With EVM write broadcasting
cre workflow simulate . --trigger-index 0 --broadcast
```
